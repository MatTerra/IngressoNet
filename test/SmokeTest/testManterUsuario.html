Test purpose

Verificar o funcionamento da página de manutenção do perfil.

Known issues:

  • Caso o usuário seja responsável por uma partida e peça pra deletar seu perfil, seu cartão será apagado, mas o usuário não.

Preconditions of the test:

  • O banco de dados deve estar rodando com o IP local 172.18.0.2 ou o IP do banco deve ser atualizado na classe MySQLHelper.
  • Rodar o programa e fazer login com o usuário 052.327.931-01 senha 123456.
  • Clicar no botão "Configurar" localizado ao lado do CPF com 3 linhas horizontais paralelas.

Test documents

  • Nenhum documento necessário

Test cases

Test case salvar sem alteração
Entrada/Procedimento
  1. Clicar no botão "Salvar" antes de alterar número do cartão e número de segurança.
Saída
  • O botão deve estar desativado, não permitindo cliques.
Test case alterar incorretamente cartão
Entrada/Procedimento
  1. Alterar número do cartão para número inválido (exemplo: 123 456 789 987.)
  2. Clicar no botão "Salvar".
Saída
  • Mensagem de Número de cartão inválido.
Test case alterar incorretamente CVV
Entrada/Procedimento
  1. Alterar CVV para número inválido, ou seja, menor que 100.
  2. Clicar no botão "Salvar".
Saída
  • Botão deve estar desativado.
Test case alterar corretamente
Entrada/Procedimento
  1. Alterar número do cartão para número válido (exemplo: 5106 4186 3286 1791.)
  2. Alterar cvv para qualquer número entre 100 e 9999.
  3. Clicar no botão salvar.
Saída
  • Mensagem de Cartão alterado com sucesso.
Test case senha antiga incorreta
Entrada/Procedimento
  1. Preencher o campo "Antiga senha" com 123455.
  2. Preencher os campos da nova senha com 123456.
Saída
  • Mensagem de Senha incorreta.
Test case senha nova igual antiga
Entrada/Procedimento
  1. Preencher o campo "Antiga senha" com 123456.
  2. Preencher os campos da nova senha com 123456.
Saída
  • Mensagem de Senha Nova igual Ã  antiga.
Test case senha nova e confirmação diferentes
Entrada/Procedimento
  1. Preencher o campo "Antiga senha" com 123456.
  2. Preencher o campo da nova senha com 1234567.
  3. Preencher o campo de confirmação da senha com 1234568.
Saída
  • Mensagem As senhas fornecidas não são iguais.
Test case alterar senha corretamente
Entrada/Procedimento
  1. Preencher o campo "Antiga senha" com 123456.
  2. Preencher os campos da nova senha com 1234567.
  3. Alterar novamente a senha para 123456.
Saída
  • Mensagem de Senha alterada com sucesso.
Test case voltar
Entrada/Procedimento
  1. Clicar no botão "Voltar".
Saída
  • O programa retorna para o Dashboard.
Test case apagar
Entrada/Procedimento
  1. ATENÇÃO: Para este caso de teste, deslogue do usuário 052.327.931-01 e acesso o usuário: 726.899.300-56 senha: 123456
  2. Clicar no botão "Apagar Perfil".
  3. Confirmar a exclusão no diálogo apresentado.
Saída
  • Mensagem de perfil apagado.
  • O programa retorna para a página de login.
