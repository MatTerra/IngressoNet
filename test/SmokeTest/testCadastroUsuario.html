Document Owner:
Bruno Murta, Mateus Berardo, Thiago Cardoso

Last Document Change:
21. 11. 2019

Status of Document:
Standard

Test purpose

Verificar o funcionamento do cadastro de usário.

Known issues:

  • Não existem problemas conhecidos

Preconditions of the test:

  • O banco de dados deve estar configurado com o IP local 172.18.0.2 ou o IP do banco deve ser atualizado na classe MySQLHelper.
  • Rodar o programa e clicar em "Não possuo cadastro" para abrir a tela de cadastro.
  • Não inicie o banco de dados enquanto não for solicitado.

Test documents

  • Nenhum documento é necessário.

Test cases

Test case banco de dados não iniciado
Entrada/Procedimento
  1. Garanta que o programa não tem acesso ao banco de dados.
  2. Preencha o CPF com o valor 000.000.000-00
  3. Preencha o número do cartão com 5555666677778884
  4. Preencha o número de segurança com 123
  5. Digite "123456" no campo senha
  6. Digite "123456" no campo repita a senha
  7. Clique em cadastrar.
  8. Inicialize o banco de dados
Saída
  • Um diálogo avisando que houve um erro será exibido. Erro 01 - No Connection to DB.
Test case mensagens de erro
Entrada/Procedimento
  1. Clique em cadastrar.
Saída
  • Texto "CPF Inválido" em vermelho substitui o CPF.
  • Texto "A senha deve ter ao menos 6 dígitos!" em vermelho substitui a senha.
  • Texto "Cartão Inválido" em vermelho substitui o nú do cartão.
  • Texto "Número de Segurança Inválido" em vermelho substitui o número de segurança.
Test case pré-validação CPF
Entrada/Procedimento
  1. Digite "abc.!" no campo CPF
  2. Digite 11 0's no campo CPF
Saída
  • O texto do campo CPF deve estar vermelho durante a inserção dos caracteres.
  • O campo CPF não deve aceitar a inserção de letras ou caracteres especiais.
  • O texto final deve ser "000.000.000-00" em branco
Test case CPF inválido
Entrada/Procedimento
  1. Preencha o CPF com o valor 000.000.000-01
  2. Preencha a senha e a repetição da senha com 123456
  3. Preencha o número do cartão com 5555666677778884
  4. Preencha o número de segurança com 123
  5. Clique em cadastrar.
Saída
  • Texto "CPF Inválido" em vermelho substitui o CPF.
Test case pré-validação senha
Entrada/Procedimento
  1. Digite "abc.!" no campo senha
  2. Digite "123456" no campo senha
  3. Deselecione o campo senha clicando em qualquer outro campo
Saída
  • O texto do campo senha deve estar vermelho até a inserção do número 6.
  • O texto final deve estar em branco
  • O campo não deve permitir a inserção de letras ou caracteres especiais.
  • Ao deselecionar o campo, o texto deve ser ocultado.

Test case cadastro bem sucedido
Entrada/Procedimento
  1. Garanta que o banco de dados está acessível.
  2. Preencha o CPF com o valor 726.899.300-56
  3. Preencha o número do cartão com 5555666677778884
  4. Preencha o número de segurança com 123
  5. Digite "123456" no campo senha
  6. Digite "123456" no campo repita a senha
  7. Clique em cadastrar.
  8. Acesse a tela de manutenção do perfil e apague o usuário após o teste.
Saída
  • Um diálogo avisando que o cadastro foi efetuado com sucesso será exibido.
  • Após confirmação no diálogo, o programa retorna para a tela de login.
Test case banco de dados caído
Entrada/Procedimento
  1. Encerre o banco de dados
  2. Preencha o CPF com o valor 000.000.000-00
  3. Preencha o número do cartão com 5555666677778884
  4. Preencha o número de segurança com 123
  5. Digite "123456" no campo senha
  6. Digite "123456" no campo repita a senha
  7. Clique em cadastrar.
Saída
  • Um diálogo avisando que houve um erro será exibido.
  • Caso os outros testes tenham sido executados na mesma instância do programa, o erro será Erro 02 - Erro na Query.
  • Caso nenhum acesso a banco de dados tenha sido feito antes, o erro será Erro 01 - No Connection to DB.
